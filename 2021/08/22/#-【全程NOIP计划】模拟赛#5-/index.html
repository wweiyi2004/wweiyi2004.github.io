【全程NOIP计划】模拟赛#5

A 么么题

题目描述

nnn 个人坐成一圈打牌，第 iii 个人开始时手中有 aia_iai​ 张牌。

从第一个人开始，每个人轮流选择出牌。轮到某个人选择出牌时，他可以选择出一张牌，也可以选择不出牌。不过，如果上一个出牌的人是他自己，或者从来没有人出过牌，那么他就必须选择出牌。然后轮到他的下一个人选择出牌，nnn 的下一个人是 111。如果有一个人出完了他所有的牌，游戏结束。

你需要求出，游戏最多能进行多少回合。一个人一次选择叫做一回合。

输入格式

第一行包括一个整数 nnn。

第二行 nnn 个整数，第 iii 个表示 aia_iai​。

输出格式

输出一个整数表示答案

输入输出样例

输入 #1

输出 #1

输入 #2

输出 #2

输入 #3

输出 #3

说明/提示

样例一解释：

  • 第一回合 111 必须出牌。
  • 第二回合 222 选择出牌。
  • 第三回合 111 选择不出牌。
  • 第四回合 222 必须出牌，他的牌出完了，游戏结束。

共进行了 444 回合，容易发现没有回合数更多的方案。

对于前 30%30\%30% 的数据，n=2n=2n=2，ai≤10a_i\le 10ai​≤10。

对于前 60%60\%60% 的数据，n≤10n\le 10n≤10，ai≤10a_i\le 10ai​≤10。

对于前 95%95\%95% 的数据，n≤105n\le 10^5n≤105，ai≤104a_i\le 10^4ai​≤104。

对于全部数据，2≤n≤1052\le n\le 10^52≤n≤105，1≤ai≤1091\le a_i\le 10^91≤ai​≤109。

思路

最优策略下每个人一定都是连续出牌，因为“A 出牌 → 其他人出牌 → A 出牌”一定不如“A 出牌 → 其他人不出 → A 出牌 → 其他人出牌”。而我们又想让不出牌的次数尽可能多，所以最优的方案应该是按 1,n,n−1,n−2…1,n,n-1,n-2\dots1,n,n−1,n−2… 的顺序连续出牌，每个人连续出到剩一张为止，由它前面最近的多余一张牌的人接手，如果没人接手就他自己出完。按照这个思路模拟即可。

当然你也可以接着推式子，但是没必要。推完之后是答案是 n(sum−n)−x+1n(sum-n)-x+1n(sum−n)−x+1，其中 xxx 是从最后一个出牌的人到 111 的距离。标程是式子做法。

  • 如果你挂样例三：注意有 a2=1a_2=1a2​=1 的情况，最后一个出牌的不一定是 222。
  • 如果你挂 #1：你可能没处理好 a1=1a_1=1a1​=1 的情况，应该特判输出 111。
  • 如果你挂 #20：最后一个点是要爆 long long 的，需要用 unsigned long long 输出。

B 迷迷题

题目描述

环形 01 串是一种首尾相接的字符串，其中每个字符都是 0 或者 1。

取出环形 01 串中连续的一段称为子串。显然，对于一个长度为 nnn​ 的环形 01 串，它有 nnn​ 个长度为 kkk​ 的子串，其中 k≤nk \le nk≤n​。

而长度为 kkk 的 01 串共有 2k2^k2k 种。如果对于一个环形 01 串，它有全部 2k2^k2k 种长度为 kkk 的子串，那它就是完美的。

请构造出长度尽可能小的、完美的环形 01 串。

输入格式

只有一个整数 kkk

输出格式

输出一行你构造出来的环形 01 串。如果有多种方案，输出任意一种即可。

输入输出样例

输入 #1

输出 #1

说明/提示

样例解释：这个环形串包含了四个长度为 222 的子串：11，10，00，01，因此它是完美的。显然没有更短的方案。

本题共有 202020 个测试点，其 kkk 值分别为 111 到 202020，每个测试点满分 555 分。本题使用 SPJ 给分，具体方式如下：

温馨提示：建议选手赛时自己写一个 checker。（如果您能一遍过当我没说）

思路

由于 SPJ 的赋分方式，各种乱搞都能拿分，这里不一一介绍。我自己写的随机化乱搞得到了 62 分的好成绩。

前 nnn 个字符放 0，状压记录已经出现的长度为 nnn 的子串，然后贪心能放 1 就放 1，会出现重复就放 0，这样前 2n2^n2n 个字符就是最短的串。

证明：首先，如果这样一直放，直到放不了了停止，那么串的末尾一定是 n−1n-1n−1 个 0，就像这样：000000111111…………00000[放不了]
这个结论可以反证：假如最后是 000000111111…………XXXXX[放不了]，那么 XXXXX 一定出现了第三遍，那么 0XXXXX 和 1XXXXX 中至少有一个是重复的，这个串本该在这之前就放不了了。除非 XXXXX 就是最开头的 00000，因为这前面没有字符。

于是我们把 000000111111…………00000 最后的 00000 删掉，这就是一个天然形成的环形串，这其中没有重复的子串。那么要证明的就是，每个子串都在这其中出现了。由于我们是贪心放 1，所以只要 ABCDE0 出现，ABCDE1 就一定出现。考虑这前面的字符，0ABCDE 和 1ABCDE 也都一定出现。换句话说，只要一个末尾是 0 的串出现了，那么去掉末尾这个 0，在开头加上任意字符，得到的串也出现了。

我们发现最开始钦定的 nnn 个字符是 0，那么去掉末尾这 nnn 个 0，在开头加上 nnn 个任意字符……这正是所有长度为 nnn 的串！

欧拉回路也能做，考虑把所有长度为 n−1n-1n−1 的串当作点，长度为 nnn​ 的串当作边，本质上就是在原图找一个欧拉回路。

C 喵喵题

题目描述

一个环上有 nnn 个点，点分为黑点和白点，初始全是白点。你可以选择任意多次操作，选择这个环上长度为 kkk 的连续一段，然后让这一段中所有点颜色翻转，即黑变白，白变黑。环不可以旋转。求能达到多少种不同的状态。由于答案非常大，你只需要输出用二进制表示这个数有多少位数（答案用十进制输出）

输入格式

一行两个整数 nnn 和 kkk。

输出格式

一个整数表示答案

输入输出样例

输入 #1

输出 #1

说明/提示

样例解释：容易发现可以达到所有黑点个数为偶数的状态，共 888 种。888 用二进制表示为 100010001000，是 444 位数。

对于 30%30\%30% 的数据，n≤20n\le 20n≤20。

对于 60%60\%60% 的数据，n≤1000n\le 1000n≤1000。

对于另外 20%20\%20% 的数据，k=4k=4k=4。

对于全部数据，1≤k≤n≤1091\le k\le n\le 10^91≤k≤n≤109。

思路

打表找规律数学题。（写暴力打表，大眼观察出结论）

结论：状态数是 2n−gcd⁡(n,k)+(kgcd⁡(n,k) mod 2)2^{n - \gcd(n,k) + (\frac{k}{\gcd(n,k)} \bmod 2)}2n−gcd(n,k)+(gcd(n,k)k​mod2)，于是答案是 n−gcd⁡(n,k)+(kgcd⁡(n,k) mod 2)+1n - \gcd(n,k) + (\frac{k}{\gcd(n,k)} \bmod 2) + 1n−gcd(n,k)+(gcd(n,k)k​mod2)+1。

正经证明：由于这本质上是一个异或线性空间，所以状态数一定是 222 的线性基的元素个数次幂，二进制位数就等于这东西 +1+1+1。于是 606060 分给了 bitset 线性基。由于线性空间的性质，线性基的每个元素都可以异或其它的元素而保持线性空间不变，于是把题面换成“可以翻转固定的一段长度为 kkk 的段，可以翻转任意距离为 kkk 的两点”，答案是相同的。

如果你不知道线性基是啥，也没关系。你只需要明白这个转化是等价的就可以。这个很容易证明：翻转距离为 kkk 的两点就是翻转相邻两个连续段，翻转一个连续段就是翻转固定的那个连续段，再用两点把它“移动”到需要的位置。（当然，做完这题之后最好去学习一下：P3812 【模板】线性基）

那我们先不考虑那个固定的连续段，只考虑距离为 kkk 的两点。发现它其实是 gcd⁡(n,k)\gcd(n,k)gcd(n,k) 个相互独立的环，每个环 ngcd⁡(n,k)\frac{n}{\gcd(n,k)}gcd(n,k)n​ 个元素，原环上距离为 kkk 的两点就等于一个独立环上的相邻两点。由于一个独立环上可以达到的状态是所有偶数个黑点的状态，所以是 2ngcd⁡(n,k)−12^{\frac{n}{\gcd(n,k)}-1}2gcd(n,k)n​−1，gcd⁡(n,k)\gcd(n,k)gcd(n,k) 个环就是 2n−gcd⁡(n,k)2^{n-\gcd(n,k)}2n−gcd(n,k)。如果你直接把这个当答案会得到 505050 分的好成绩。

那么那个固定的连续段到底有没有贡献呢，显然如果它可以被距离为 kkk 的两点表示出来就没有贡献。由于它在每个独立环内都是平等的，所以每个独立环内有 kgcd⁡(n,k)\frac{k}{\gcd(n,k)}gcd(n,k)k​ 个它的元素，所以只要它是偶数就没有贡献，是奇数就有贡献，有贡献就会让能达到的状态翻倍。

于是就得到了一开始的式子：2n−gcd⁡(n,k)+(kgcd⁡(n,k) mod 2)2^{n - \gcd(n,k) + (\frac{k}{\gcd(n,k)} \bmod 2)}2n−gcd(n,k)+(gcd(n,k)k​mod2)​。

D 萌萌题

题目描述

滴叉想要成为管理员，但是要成为管理员，就必须先打败所有的管理员。

有 nnn 个管理员，第 iii 个管理员的初始体重是 aia_iai​，还有一个团结度 bib_ibi​。由于管理员有时候会休假，所以现在只有前 kkk 个管理员在任。滴叉计划在 kkk 天之内打败这 kkk 名管理员。滴叉每天可以选择一个没被打败过的管理员，如果此时滴叉的体重大于等于对方的体重，就可以将对方打败。在第 iii 个管理员被打败之后，其他管理员会团结起来，导致所有管理员的体重都增加了 bib_ibi​。滴叉的体重永远不变。

滴叉会以最优策略选择打败管理员的顺序，确保自己能获胜。但是滴叉不知道 kkk 的值，所以你需要对于所有 k∈[1,n]k\in [1,n]k∈[1,n]，计算出为了打败前 kkk 个管理员，滴叉的体重至少是多少。

输入格式

第一行输入一个整数 nnn。

接下来 nnn 行，每行两个整数，第 i+1i+1i+1 行表示 aia_iai​ 和 bib_ibi​。

输出格式

输出 nnn 行，第 iii 行对应 k=ik=ik=i 时的答案

输入输出样例

输入 #1

输出 #1

输入 #2

输出 #2

输入 #3

输出 #3

说明/提示

样例一解释：当 k=4k=4k=4 时候，滴叉的体重为 333，滴叉可以这样做：

  • 打败 111，管理员的体重变成了 0,1,2,30,1,2,30,1,2,3。
  • 打败 222，管理员的体重不变。
  • 打败 444，管理员的体重变成了 1,2,3,41,2,3,41,2,3,4。
  • 打败 333。

如果滴叉的体重小于等于 222，就无法打败所有管理员。所以 k=4k=4k=4 时输出 333。

对于前 20%20\%20% 的数据，n≤10n\le 10n≤10。

对于前 50%50\%50% 的数据，n≤20n\le 20n≤20。

对于前 70%70\%70% 的数据，n≤3000n\le 3000n≤3000。

对于全部数据，1≤n≤5×1051\le n\le 5\times 10^51≤n≤5×105，−109≤ai≤109-10^9\le a_i\le 10^9−109≤ai​≤109，−109≤bi≤109-10^9\le b_i\le 10^9−109≤bi​≤109。

在本题中，体重可能是负数，这并不影响题意。

请使用较快速的读入方式。

思路

考虑逆向枚举打败顺序，维护一个 xxx 表示此时你的最小体重，初始时是负无穷。那么逆向打败一个管理员就相当给 xxx 加上 bib_ibi​，然后和 aia_iai​ 取 max。于是你知道了顺序就可以求出答案，部分分 202020 分给枚举全排列，505050 分给状压。

如果你暴力输出了打败的顺序，就可以发现，新添加一个管理员的时候，原有的顺序不会被破坏。这暗示着有一种“固有顺序”，只要有了 aaa 和 bbb 就可以判断出管理员之间的顺序。由于两次先加后 max 操作是可以合并的，我们考虑合并 max⁡(ai,x+bi)\max(a_i,x+b_i)max(ai​,x+bi​) 和 max⁡(aj,x+bj)\max(a_j,x+b_j)max(aj​,x+bj​)。如果先进行 iii，那就是 max⁡(max⁡(ai+bj,aj),x+bi+bj)\max(\max(a_i+b_j,a_j),x+b_i+b_j)max(max(ai​+bj​,aj​),x+bi​+bj​)，先进行 jjj 就是 max⁡(max⁡(aj+bi,ai),x+bi+bj)\max(\max(a_j+b_i,a_i),x+b_i+b_j)max(max(aj​+bi​,ai​),x+bi​+bj​)。区别只有 max⁡(ai+bj,aj)\max(a_i+b_j,a_j)max(ai​+bj​,aj​) 和 max⁡(aj+bi,ai)\max(a_j+b_i,a_i)max(aj​+bi​,ai​)，如果前者小于后者就应该 iii 在前面，否则 jjj 在前面。注意我们讨论的是打败的逆序，所以“在前面”等于“后打败”。于是可以把这个东西当作 cmp 函数丢进 sort 里。如果你直接这么做，会发现过不了样例三。发生什么事了？

问题在于，这个 cmp 如果分不出两个元素的顺序，仅仅意味着它们挨在一起的时候不分顺序，不代表它们什么时候都不分顺序。假如它们中间还有其他的元素，我们把这中间所有的元素都合并成一个，记作 aka_kak​ 和 bkb_kbk​。那么 i,j,ki,j,ki,j,k 三个元素的合并有以下情况：

  • 假如是 i,k,ji,k,ji,k,j 的顺序，合并后的 a=max⁡(ai+bk+bj,ak+bj,aj)a=\max(a_i+b_k+b_j,a_k+b_j,a_j)a=max(ai​+bk​+bj​,ak​+bj​,aj​)。
  • 假如是 j,k,ij,k,ij,k,i 的顺序，合并后的 a=max⁡(aj+bk+bi,ak+bi,ai)a=\max(a_j+b_k+b_i,a_k+b_i,a_i)a=max(aj​+bk​+bi​,ak​+bi​,ai​)。

由于 i,ji,ji,j 按照之前的 cmp 分不出顺序，所以 max⁡(ai+bk+bj,aj)=max⁡(aj+bk+bi,ai)\max(a_i+b_k+b_j,a_j)=\max(a_j+b_k+b_i,a_i)max(ai​+bk​+bj​,aj​)=max(aj​+bk​+bi​,ai​)，也就是说区别只在中间那个数。显然应该选一个 bbb 小的放在后面，大的放在前面。于是第二关键字应该按照 bbb 从大到小排序。（然而貌似第二关键字从小到大排 xxx 还是从大到小排 xxx 都 AC 了，对于这些东西的正确性我不确定）

并且通过大力分类讨论，这个新的 cmp 满足 sort 内比较函数的所有需求（小于号传递性，等号传递性）。那么处理前 kkk 个元素就是单点插入，全局查询，由于这个东西是区间可合并的，于是上一个线段树就做完了。
